# ImageMatch

一个基于 MFC + OpenCV 的桌面图像配准与拼接工具，面向大分辨率图像场景，支持多种刚性配准方法、配准日志输出、结果拼接，以及特征点和匹配关系可视化。

## 功能概览

- 支持加载 `Source` 和 `Target` 两张图像，支持拖拽导入。
- 支持多种配准方法：
  - `PHASE`：基于相位相关的快速平移配准。
  - `ECC`：基于强度优化的非特征点刚性配准。
  - `ORB`：轻量级特征点配准。
  - `AKAZE`：鲁棒特征点配准。
  - `SIFT`：更强鲁棒性的特征点配准。
- 支持输出刚性变换矩阵、旋转角、平移量、RMSE、关键点数、匹配数、内点数等结果。
- 支持大图加速：针对超大分辨率图像自动降低工作分辨率、限制特征点数量与匹配数量。
- 支持特征点结果可视化：
  - 在 `Source` 窗格显示源图关键点和目标点反投影连线。
  - 在 `Target` 窗格显示目标图关键点和源点正向投影连线。
- 支持拼接结果预览，并提供缩放、平移查看能力。

## 大图优化策略

项目针对高分辨率图像做了专门优化，避免直接在原始分辨率下执行高成本配准：

- 自动识别大图与超大图场景。
- `PHASE`：
  - 使用更激进的降采样，优先快速估计平移。
- `ECC`：
  - 使用相位相关结果作为初值。
  - 在大图模式下降低工作尺寸与最大迭代次数。
- `ORB / AKAZE / SIFT`：
  - 自动降低工作图尺寸。
  - 自动限制特征点数量。
  - 自动限制保留匹配对数量。

对于超大图，推荐优先尝试：

1. `PHASE`
2. `ECC`
3. `ORB`

如果图像纹理复杂且需要更强鲁棒性，再尝试 `AKAZE` 或 `SIFT`。

## 日志输出

每次配准都会输出详细日志，显示在结果文本框中，同时通过 `TRACE` 输出到调试窗口。日志包含：

- 输入图像尺寸
- 加速策略说明
- 工作分辨率与缩放比例
- 灰度转换耗时
- 特征提取耗时
- 匹配与 RANSAC 耗时
- ECC / 相位相关耗时
- RMSE 计算耗时
- 拼接耗时
- 总耗时

便于分析不同算法在大图下的速度表现。

## 界面说明

- `Source`：源图窗口
- `Target`：目标图窗口
- `Stitched`：拼接结果窗口
- 方法按钮：
  - `PHASE`
  - `ORB`
  - `AKAZE`
  - `SIFT`
  - `ECC`
- `Reset`：重置当前视图缩放与平移

交互方式：

- 鼠标左键拖拽：平移图像
- 鼠标滚轮：围绕光标缩放
- 拖拽文件到 `Source` 或 `Target` 窗口：直接加载图像

## 构建环境

- Windows

工程文件：

- 解决方案：`ImageMatching.sln`
- 项目文件：`ImageMatching.vcxproj`

## 使用流程

1. 加载或拖拽源图和目标图。
2. 点击一种配准方法。
3. 查看结果框中的：
   - 配准状态
   - 耗时
   - 工作缩放比例
   - 关键点/匹配/内点数量
   - 刚性变换矩阵
   - 详细日志
4. 若使用特征点方法，可在 `Source` / `Target` 窗格中查看特征点和匹配连线。
5. 在 `Stitched` 窗格中查看拼接结果。

## 当前实现说明

- 当前配准模型以刚性/欧式变换为主，适用于平移加小角度旋转场景。
- `PHASE` 和 `ECC` 属于非特征点方法，适合速度优先场景。
- `ORB / AKAZE / SIFT` 属于特征点方法，适合纹理较丰富、需要更强匹配鲁棒性的场景。
